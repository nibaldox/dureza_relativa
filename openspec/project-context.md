## Project Summary

`dureza_relativa` is a bilingual (UI Spanish, code-mixed) hardness-classification app for
drilling wells. It ingests CSV files with timestamped drill intervals, derives duration
metrics, assigns hardness labels, and renders interactive visualizations.

The project ships two parallel front-ends that must stay behaviorally equivalent:

| Layer       | Stack                                     | Entry point                |
| ----------- | ----------------------------------------- | -------------------------- |
| Python app  | Streamlit + pandas + Plotly               | `streamlit_app.py`         |
| Web app     | React 18 + TypeScript + Vite + Plotly.js  | `webapp/src/main.tsx`      |

## Domain Rules (from `data_processor.py`)

- Required CSV columns: `tiempo inicio`, `tiempo final`.
- Optional columns: `este`, `norte`, `elevacion`, `drill_pattern`.
- Derived columns: `duracion` (minutes), `dureza` (categorical: roca suave / media /
  dura / muy dura), `indice_dureza` (continuous 0..100).
- Categories — per `EJEMPLO_README.md`:
  - roca suave: < 16 min
  - roca media: 16..24 min
  - roca dura: 24..40 min
  - roca muy dura: > 40 min

2. **Internet access**: README explicitly warns that `npm install` may fail in offline

   independently. Without parity tests, regressions can diverge silently.

5. **Hardcoded logging**: `data_processor.py` configures logging at import time, which
   makes library use awkward. SDD work on the classification layer should consider
   moving this to the entry point only.
